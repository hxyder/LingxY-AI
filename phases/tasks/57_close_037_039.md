## Bug #1：连续翻译不同段落显示上次内容（UCA-037）

### 根因
`browser_ext/content_script/selection-cache.js` 中 selection state 使用可变全局变量，frame closure 捕获的是 stale reference。

### 修复方案：Selection Snapshot Model

```js
// selection-cache.js

let _pendingRequestId = null;

export function captureSnapshot(text, rect, url) {
  const requestId = crypto.randomUUID();
  _pendingRequestId = requestId;
  return { text, rect, url, requestId, timestamp: Date.now() };
}

export function isCurrentRequest(requestId) {
  return requestId === _pendingRequestId;
}
```

```js
// 调用处（runQuickAction）
const snapshot = captureSnapshot(selectedText, selectionRect, location.href);
const result = await fetchTranslation(snapshot.text);

// 响应回来时验证 requestId 是否仍然有效
if (!isCurrentRequest(snapshot.requestId)) {
  return; // 用户已选择了新的文本，丢弃旧结果
}
showInlineResult(result);
```

**关键文件**：
- `browser_ext/content_script/selection-cache.js`
- `browser_ext/content_script/inline-result-frame.js`（或对应的 content script）

## Bug #2：追问网络搜索返回旧内容（UCA-039）

### 根因
依赖 UCA-054 的 tool observation injection 修复（tool result 未注入回下一轮 prompt）。
额外问题：LLM 有时不调用 `web_search_fetch` 直接从记忆回答。

### 修复方案

1. **强制搜索**：当 TaskSpec `needs_current_web_data = true` 时，system prompt 注入：
   ```
   REQUIRED: You MUST call web_search_fetch before answering. 
   Do NOT answer from memory for current-events questions.
   ```

2. **搜索验证**：tool_using executor 完成后，检查 transcript 中是否有 `web_search_fetch` call：
   ```js
   const searchWasCalled = transcript.some(t => t.tool_name === "web_search_fetch");
   if (taskSpec.needs_current_web_data && !searchWasCalled) {
     // 强制降级为 partial_success，提示用户可能信息不新
     result.status = "partial_success";
     result.note = "[UCA] 未能确认最新数据，结果可能不是最新的";
   }
   ```

**关键文件**：
- `src/service/core/executor/tool_using/agent-loop.mjs`（依赖 UCA-054）
- `src/service/core/task-runtime.mjs`（搜索验证逻辑）
