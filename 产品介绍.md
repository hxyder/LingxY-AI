# 灵犀 LingxY · 产品介绍

> 一个常驻桌面、随叫随到的 AI 助手。
> 你说一句话，它读懂当前窗口、调用合适的工具、把事情做完。

灵犀不是一个聊天框，而是一个能"看到你在做什么、知道该用哪个工具、把结果交给你"的桌面工作伙伴。

数据本地、配置本地、模型选你自己的，AI 调用走你自己的 API key。没有账号注册、没有云端订阅、没有数据回传。

---

## 一句话能做的事

下面这些用户输入，灵犀会自动选合适的工具完成，不需要你预先配置工作流：

```
今天有什么重要邮件，整理给我
帮我把这个 PDF 的关键信息提取成表格
打开微信
分析这个网页的论点
每天早上 9 点把昨天的会议纪要整理成日报
帮我截个图，分析里面的数据
把这段中文翻译成日文
搜索一下最近 GPT 的发布消息，整理成 PPT
```

它会判断哪些步骤需要联网、哪些需要打开文件、哪些需要让你确认（比如发邮件之前），然后逐步推进。

## 功能详细清单

### 1. 当前窗口理解

灵犀会尽量把"你正在看的东西"变成任务上下文：

- 浏览器：读取当前页面 URL、标题、页面来源信息，必要时再抓取正文。
- 文件：识别资源管理器或拖入的本地文件路径，按任务需要读取内容。
- Office：通过 Word / Excel / PowerPoint 加载项获取选中内容、文档上下文或文件路径。
- 剪贴板：可读取你刚复制的文字、路径或片段。
- 截图：可以截取当前屏幕或窗口，让 Vision 模型分析图表、报错、表格或界面。
- 会话历史：如果你在一个 conversation 里继续追问，它会带上前文任务和结果。

它不会因为你给了文件就自动打开文件；只有"总结/读取/分析/修改这个文件"这类任务才需要读文件。像"结合我的简历搜索岗位"会把简历当本地材料，把岗位信息当外部信息需求。

### 2. 模型与执行器

你可以按任务选择模型和执行方式：

- 快速回答：走轻量 fast executor，适合改写、解释、翻译、短问答。
- 工具任务：走 tool-using executor，模型可以调用搜索、文件、文档、邮件、日程等工具。
- 深度规划：走 agentic executor，适合多步骤研究、资料整理、跨工具工作流。
- 图片任务：走 multimodal / vision 路径，能读截图、图片、PDF 页面渲染图。
- 本地模型：可接 Ollama，把敏感或离线任务交给本地模型。
- CLI agent：可接 Claude Code / Codex / Kimi CLI，让代码类任务交给专门 CLI 执行。

支持的主要 provider 包括 Anthropic、OpenAI/兼容 API、DeepSeek、Kimi/Moonshot、xAI、Mistral、Ollama 以及代码 CLI。API key 由用户自己提供，灵犀不做代理服务器。

### 3. 工具调用

灵犀内置的工具不是装饰，而是任务执行路径的一部分：

- 搜索网页：搜索当前信息、抓取网页正文、保存来源证据。
- 文件操作：读文件、写文件、列目录、找最近文件、打开文件、在资源管理器中定位。
- 文档生成：生成 Word、Excel、PPT、PDF/HTML 报告和 Mermaid 图。
- 应用启动：打开 Excel、Word、浏览器、会议软件、微信等本地应用；如果有歧义，会要求选择。
- 剪贴板：读取或写入剪贴板。
- 截图分析：截图后交给 Vision 模型解释。
- 本地脚本：在审批后运行 PowerShell / Node / Python。
- 系统通知：任务完成、失败、等待审批、定时任务触发时弹通知。
- 定时任务：创建、列出、暂停、恢复、手动运行、删除 schedule。
- Connector workflow：通过 Gmail / Outlook / Calendar / Drive / OneDrive 等连接器执行草稿、发送、创建事件、读取资料等动作。
- MCP / 插件：接入外部 MCP server 或插件提供的工具。

每次工具调用都会进入任务 timeline：模型为什么调用、传了什么参数、工具返回什么、是否成功、有没有产出文件。出错时可以在 Console 里追踪，不靠模型自述猜。

### 4. 文档与交付物

灵犀可以生成真正的本地文件：

- PPTX：研究报告、方案汇报、技术摘要、产品介绍。
- DOCX：会议纪要、日报、周报、分析报告、邮件草稿。
- XLSX：发票/图片/网页/论文信息提取成表格，支持对比表和结构化数据。
- PDF / HTML：适合导出、打印、分享的报告。
- Mermaid：流程图、时序图、架构图、状态机图。
- 截图与中间文件：保存在任务输出目录，方便复查。

Preview 窗口用于显示生成过程和结果，不需要等任务全部结束才知道它在做什么。

### 5. 邮件、日历和网盘

连接账户后，灵犀可以处理常见办公流：

- 读取最近邮件，筛选重要邮件，生成晨间摘要。
- 起草回复、整理收件人和主题、准备正文。
- 通过 Gmail / Outlook / SMTP 发送邮件，但高风险发送默认需要确认。
- 读取 Microsoft / Google Calendar，整理会议、生成周报或待办。
- 在 Drive / OneDrive 工作流里查找、上传或整理文件。
- 把定时任务和邮件/日历连接起来，比如每天早上整理邮件、每周五生成周报。

核心原则是：AI 可以准备，用户负责拍板。尤其是发邮件、上传、创建日程这类副作用动作，会进入确认卡。

### 6. 定时任务与自动化

灵犀的 schedule 不是简单提醒器，而是能唤醒完整 AI 任务的后台系统：

- `cron`：每天、每周、每月、工作日等重复任务。
- `at`：只执行一次的绝对时间任务，执行后自动完成。
- `interval`：每隔 N 秒/分钟/小时运行。
- `file_watch`：文件夹或文件变化时运行。
- `clipboard_watch`：剪贴板变化时运行。

支持：

- 自然语言创建："每天 9 点整理邮件"、"明天下午 3 点提醒我"。
- 错过补跑：程序没开时错过的 cron，可按策略补跑一次、全补或跳过。
- 手动 Run now：完成或暂停的任务也可以手动再跑，不会偷偷重新启用自动运行。
- 审批：定时触发后如果要发邮件、运行脚本、写文件，仍可要求用户确认。

### 7. 任务、会话和笔记

灵犀会把工作留下可追踪记录：

- Tasks：每个任务有状态、时间线、工具调用、产物、失败原因。
- Conversations：多轮对话能串起来，切回旧会话继续问。
- Notes：可保存临时笔记、总结和后续处理材料。
- History：可以搜索历史任务；配置 embedding 后可做语义搜索。
- Artifacts：生成的文件和中间文件与任务绑定，Console 里能打开和预览。

这解决的是"AI 做完以后我找不到结果/不知道它做了什么"的问题。

### 8. 扩展系统

灵犀有三层扩展：

- MCP：接入社区 MCP server，例如文件系统、Brave Search、Puppeteer、Figma 等。
- Plugins：把工具、技能、应用能力打成插件，由 Console 管理启用/禁用。
- Skills：用 `SKILL.md` 保存你的常用流程和偏好。

安装 MCP 包时会走 sandbox、manifest 检测和本地 actor guard，不让"安装一个工具"变成随意修改本机环境。

### 9. 安全与隐私

安全不是后补功能，而是框架的一部分：

- 本地优先：任务、配置、日志、产物默认在本机 `%APPDATA%\UCA\`。
- 无账号服务器：没有灵犀云账号，也没有强制订阅。
- 自带 API key：你直接连接 LLM provider，灵犀不托管请求。
- 副作用审批：发邮件、删文件、上传、运行脚本、安装包等会进入确认。
- 审计日志：关键工具调用和 schedule 操作会记录。
- 隐私设置：redaction、Presenter mode、offline mode、kill switch。
- 边界测试：HTTP mutation surface、tool policy、scheduler、external call 都有 verifier 或 behavior tests。

provider/API key 已从 `runtime.json` 明文配置迁出，配置文件只保存 `apiKeyRef`；对应 key 由本地 Secret Store 管理。后续 v1.0 仍会继续把这层升级到 OS keychain / credential store。

## 五种窗口形态，对应五种使用方式

灵犀不是一个全屏应用。它给不同场景准备了不同的入口形态：

### 1. Overlay 浮窗

- **形态**：半透明小窗口，悬浮在你当前正在看的窗口之上
- **用法**：全局快捷键唤起（默认 `Alt+Space`），输入指令，回车
- **特点**：自动读取你当前的活跃窗口（浏览器 URL、文件路径、选中文字）作为上下文
- **典型场景**：边浏览边问问题、给当前 PDF 起个总结、发邮件

### 2. Dock 主入口

- **形态**：屏幕边缘的常驻按钮
- **用法**：点一下展开 Overlay
- **副功能**：显示任务运行状态、剪贴板提示、笔记录音状态


### 3. Console 控制台

- **形态**：完整应用窗口
- **用法**：管理任务历史、查看 AI 生成的文件、配置定时任务、维护笔记和项目、调整提供商和插件
- **典型场景**：每天开机看一眼"昨晚的定时任务跑得怎么样"、整理一周累积的笔记、修改提示词模板

### 4. Popup Card 通知卡

- **形态**：屏幕右上角堆叠的卡片
- **用法**：被动接收——任务完成、需要审批、调度触发时弹出
- **特点**：非阻挡式，可以钉住、关闭、自动消失。需要审批的高风险操作（发邮件、删除文件）会在这里让你确认

### 5. Preview 预览窗

- **形态**：屏幕右边缘锚定的窗口
- **用法**：流式显示 AI 正在生成的内容（文档、图表、代码）
- **特点**：实时看到 PPT 一页一页生成、Excel 一行一行写入，过程可见、出错可中断

## 核心特点

### 上下文感知，不需要你手动复制粘贴

打开浏览器 → Overlay 已经读到当前 URL；
打开 Word → 它知道文件路径；
选中一段文字 → 它直接拿到内容。

你说"翻译这个"——它知道"这个"是什么。

### 自带模型，不锁单一服务商

Anthropic Claude、OpenAI 兼容、Ollama 本地、DeepSeek、Kimi、Moonshot、xAI、Mistral 都能配。
每个任务可以选不同模型——研究类用 Sonnet、快速回答用 Haiku、敏感内容用本地 Ollama。
API key 是你的，配置存本地，灵犀不代理你的请求。

### 工具丰富，但不打扰

30+ 内置工具：文件读写、文档生成（PPTX/Word/Excel/PDF）、网页搜索、截图、剪贴板、应用启动、GUI 自动化、定时任务、邮件、语音转写。
所有副作用操作（发邮件、删文件、运行外部命令）默认走审批流——AI 想做，你来按确认。

### 插件扩展（MCP + Skills）

- **MCP 服务器**：把灵犀接到任何 MCP 兼容的工具集（文件系统、记忆库、Brave 搜索、Puppeteer、Figma 等）。可以一键安装社区 MCP server
- **Skills**：把你重复操作 3 次以上的流程保存成 SKILL.md，下次说"按我那个习惯做" AI 就照旧执行
- **MCP install**：从 npm 包名或 GitHub URL 直接安装 MCP 服务器，自动 sandbox + 探测 manifest，不需要你研究 binFile 路径

### 集成你已经在用的服务

- **Google Workspace**：Gmail（读/写邮件）、Calendar（事件）、Drive（文件）
- **Microsoft 365**：Outlook、日历、OneDrive
- **本地邮箱**：IMAP / SMTP 直连
- **浏览器**：Chrome / Edge 扩展把当前页面 URL 同步给桌面端
- **Office**：Word / Excel / PowerPoint 加载项把选中内容同步给桌面端

OAuth 流程统一走"draft → 确认 → 发送"模式——AI 写好邮件草稿，你看一眼、改一改、按发送。不会偷偷替你发出去。

### 自然语言定时任务

不需要学 cron 语法：

```
每天早上 9 点把今天的天气发到我邮箱
每周一上午整理上周的工作日报
明天下午 3 点提醒我开会
```

灵犀解析这种自然语言，转成内部 cron 表达式，自动注入正确的时区。

### 文档生成

让 AI 直接产出可交付的文件，不只是聊天记录：

- **PPTX**：自动多页布局、配图、图表
- **Word / Excel**：结构化文档和表格
- **Mermaid**：流程图、组织图、时序图（在生成的 HTML 报告里直接渲染）
- **PDF**：通过 Edge/Chrome headless 转换

文件直接保存到 `outputs/` 目录，Console 里能直接预览和打开。

### 语音输入

Echo 唤醒词模式可选启用，说"灵犀"激活；说话即转写，转写完即作为指令发送。
转写走本地 Whisper（离线可用）或云端 provider（Anthropic/OpenAI Vision）。

## 典型使用场景

### 场景 1：每天早上的信息整理

定时任务："每天 9 点整理过去 24 小时重要邮件"

灵犀会：
1. 通过 IMAP / Microsoft Graph 拉取邮件
2. 按发件人重要度 + 主题关键字筛选
3. 生成晨间摘要 PPT 或 Markdown 报告
4. 通过 Popup Card 提醒你查看

### 场景 2：会议纪要 → 日报

会议结束，把转写文字粘到 Overlay：

```
把这个会议纪要整理成日报，提取 3 个待办，发给我自己邮箱
```

灵犀：
1. 解析转写、提取要点
2. 生成 Word 格式日报
3. 提取待办，建议加到日历
4. 起草邮件，让你审核后发送

### 场景 3：浏览网页时的随手处理

打开一篇英文技术文章，按 `Alt+Space`：

```
翻译总结，重点说技术架构部分，做成 PPT
```

灵犀读到当前 URL，抓取页面内容，翻译 + 总结 + 生成 PPT，过程在 Preview 窗口实时显示。

### 场景 4：本地文件的批处理

把一个文件夹拖到 Overlay：

```
把这些发票图片提取金额，做成 Excel 表
```

灵犀：
1. 识别拖放的文件列表
2. 对每张图调用 Vision 模型提取关键字段
3. 写入 Excel
4. Preview 窗口实时显示进度

### 场景 5：重复操作变技能

你周一到周三都说过"把今天接收到的客户邮件整理成回复模板"——灵犀检测到你重复了 3 次，弹卡片：

```
检测到您重复执行了类似流程 3 次，要保存为技能"客户邮件整理"吗？
```

确认后，下次说"按客户邮件流程处理今天的邮件"，AI 就直接执行，不需要再描述步骤。

### 场景 6：研究 + 资料整理

```
搜一下最近半年关于 AI agent 的论文，找 5 篇影响力大的，做成对比表，包括方法、benchmark、限制
```

灵犀走 agentic 多轮：
- 调 web_search 多次
- 读取每篇论文（PDF 解析或网页抓取）
- 提取结构化字段
- 生成 Excel 对比表 + Word 综述

整个过程在 Preview 显示，你可以随时中断、修改、追加要求。

### 场景 7：定时执行的工作流（Schedule）

定义一次：

```
每周五下午 4 点：
1. 拉取本周 Microsoft Calendar 上所有会议
2. 整理参会人和议题
3. 生成 Markdown 周报
4. 通过 Outlook 发给我自己存档
```

之后每周五自动执行，结果通过 Popup Card 通知你。失败会重试，并在 Console → Tasks 里留下日志。

## 设计原则（为什么这么做）

### 本地优先

- 所有数据（任务历史、笔记、配置、技能、嵌入向量）存在 `%APPDATA%\UCA\`
- 没有云端账号、没有强制注册
- 配置丢了就重建，不会"账号被封"

### 隐私可控

- 用户数据不上传（除非你主动用 cloud LLM provider，那时数据流向 LLM 厂商，由你 API key 控制）
- 内置 redaction 规则（隐私设置里可配）：截图发给 LLM 前自动遮挡身份证、手机号
- "Presenter mode"：演示时自动屏蔽通知和敏感对话
- "Kill switch"：紧急情况一键停止所有出境网络请求

### API 自带、模型自选

- 不维护中间服务器，所以也不能有"单点故障"
- 你换 provider 不用换工具
- 你想用本地 Ollama 跑离线推理，全套工具仍然能用（部分需要联网的工具会被自动跳过）

### 可扩展但不强迫

- MCP / Skills / Plugins 三层扩展，但每一层都是可选的
- 默认配置开箱即用，不强迫你装一堆 npm 包
- 不熟悉的功能默认关闭（Echo 唤醒词、邮件 connector、自动调度等）

### 副作用必经审批

- 发邮件、删文件、运行外部命令、安装 MCP 包——这类高风险动作走 Popup Card 审批
- AI 起草、用户拍板，错了不会偷偷发出去

## 适合谁用

✅ **适合**：
- 每天处理大量信息（邮件、会议、文档）的知识工作者
- 想用 AI 但不想被锁定在某一个 provider 的人
- 重视本地数据 / 隐私 / 离线可用性的人
- 愿意配置自己的 API key 来换取可控性的人
- 开发者 / 工程师 / 研究者（能受益于 MCP 和工具调用）

❌ **不适合**：
- 想要"开盒即用、零配置"的纯消费者（本工具假设你有自己的 API key）
- 完全不联网的场景（LLM 调用即需网络，本工具没有"100% 离线"模式）
- 只想要单一聊天框、不需要工具调用的人（直接用 Claude Desktop 或 ChatGPT 即可）
- 团队/企业批量部署（当前是单人桌面工具，没有多用户/SSO）

| 文档生成 | PPT / Word / Excel / PDF | 无 | 无 | 无 |
| 语音 | 唤醒词 + 转写 | 无 | 无 | 无 |
| 邮件集成 | IMAP / Gmail / Outlook | 无 | 无 | 通过插件 |
| 数据 | 本地 SQLite | Anthropic 服务器 | 云同步 | 云同步 |
| 价格 | 免费（你付 LLM API 钱） | $20/月 | $20/月 | $8-16/月 |

灵犀的差异化在于：**做"桌面工作流自动化"，不只是"桌面聊天"**。它的设计假设是你已经在用某种 LLM API，灵犀负责把这个 API 接到你的工作流里。

## 快速开始

详细安装、配置、API 列表见 [README.md](README.md)。

最简流程：

```bash
git clone <repo>
cd lingxy
npm install
npm run start:runtime    # 后台服务
npm run start:desktop    # 桌面应用（另开终端）
```

首次启动会带你走 5 步引导：
1. 系统权限（剪贴板、活跃窗口）
2. 选择你的 LLM provider 并填 API key
3. 可选：连接 Google / Microsoft 账户
4. 可选：启用 MCP 服务器
5. 完成，开始使用
